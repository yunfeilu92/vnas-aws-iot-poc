# IoT 数据管道月度费用估算

> Region: eu-central-1 (Frankfurt)
> 估算日期: 2026-03-09

## 业务场景

| 参数 | 数值 |
|------|------|
| 车辆数 | 5,000 台 |
| 每车每日运行时间 | 2 小时 |
| 消息频率 | 1 条/秒/车 |
| 单条消息大小 | 128 KB |

## 数据量总览

| 指标 | 数值 |
|------|------|
| 每日消息数 | 5,000 × 7,200s = 3,600 万条 |
| 每月消息数 | ~10.8 亿条 |
| 每日数据量 | 36M × 128KB ≈ 4.3 TB |
| 每月数据量 | ~129 TB |
| 峰值吞吐（5000车同时在线） | 625 MB/s |

## 架构

```
车端遥测数据 (128KB, 1msg/s)
  → MQTT publish: dt/vehicles/{vin}/telemetry
  → IoT Core Rules Engine
  → Amazon MSK (Kafka)
  → Managed Flink
  → S3

车端状态 (< 8KB, 低频)
  → Device Shadow (Named Shadow)
  → 在线状态、固件版本、配置同步等

查询层
  → Amazon Athena 查询 S3 数据
```

### 架构说明

- **不使用 Device Shadow 传输遥测数据**：经典 Shadow 文档上限 8KB，Named Shadow 同样受限，无法承载 128KB 消息
- **MQTT 主题发布**：IoT Core MQTT 最大 payload 128KB，恰好满足需求
- **MSK 使用 Provisioned 集群**：峰值 625 MB/s 超过 MSK Serverless 上限（200 MB/s）

## 费用明细

### 1. AWS IoT Core — ~$28,600/月

IoT Core 按 **5KB 分块计费**，128KB 消息 = 26 个计费单元。

| 子项 | 计算 | 费用 |
|------|------|------|
| Connectivity | 18M 连接分钟 × $0.08/M | ~$2 |
| Messaging | 10.8 亿消息 × 26 blocks = 280.8 亿计费消息 | ~$20,200 |
| Rules Engine (触发) | 280.8 亿 × $0.15/M | ~$4,200 |
| Rules Engine (Action) | 280.8 亿 × $0.15/M | ~$4,200 |

**Messaging 分层计价：**

| 层级 | 消息量 | 单价 | 费用 |
|------|--------|------|------|
| 0 - 250M | 250M | $1.00/M | $250 |
| 250M - 5.25B | 5B | $0.80/M | $4,000 |
| 5.25B+ | 22.83B | $0.70/M | $15,981 |
| **合计** | **28.08B** | | **$20,231** |

### 2. Amazon MSK (Provisioned) — ~$8,100/月

**Broker 选型依据：** RF=3 时，集群总写 I/O = 625 × 3 = 1,875 MB/s，加上 Consumer 读取 625 MB/s，集群总 I/O 约 2,500 MB/s。每 broker 需承载 ~278 MB/s，kafka.m5.2xlarge（基线 ~312 MB/s）满足需求。

| 子项 | 计算 | 费用 |
|------|------|------|
| 实例 | 9× kafka.m5.2xlarge × 720h × $0.65/h | ~$4,200 |
| 存储 | 3天保留 × RF3 ≈ 40TB × $0.10/GB | ~$3,900 |

### 3. Managed Flink — ~$2,500/月

| 子项 | 计算 | 费用 |
|------|------|------|
| KPU | ~32 KPUs × $0.11/h × 720h | ~$2,500 |

### 4. S3 存储 — ~$1,600 - $3,200/月

| 场景 | 数据量 | 费用 |
|------|--------|------|
| Flink 输出 Parquet（~50%压缩） | ~64 TB | ~$1,600 |
| 原始格式存储 | ~129 TB | ~$3,200 |

S3 Standard 单价：$0.0245/GB-month

### 5. 跨 AZ 数据传输 — ~$1,500 - $2,000/月

MSK broker 分布在 3 个 AZ，Flink 消费时约 2/3 的读取为跨 AZ 流量，$0.01/GB。

### 6. Amazon Athena — ~$50 - $500/月

$5/TB 扫描，具体取决于查询频率。使用 Parquet + 分区可大幅降低扫描量。

## 优化建议

| 优化方向 | 预期效果 |
|----------|---------|
| **压缩消息到 5KB 以内** | IoT Core 费用从 ~$28K 降至 ~$1.1K（降 96%） |
| **车端批量聚合**（10s 一批） | 消息数降 10 倍，IoT Core + Rules Engine 同步下降 |
| **错峰发送** | 降低峰值吞吐，MSK 可用更小规格 |
| **S3 用 Parquet 格式** | 存储减半，Athena 查询成本大幅降低 |
| **Athena 分区策略** | 按日期/车辆分区，减少每次查询扫描量 |
