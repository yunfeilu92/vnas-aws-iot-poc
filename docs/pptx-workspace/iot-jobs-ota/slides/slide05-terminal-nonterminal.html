非终态 vs 终态

Non-Terminal States (可转换)

QUEUED

Job 创建时自动设置，设备在队列中等待执行。可转换到 IN_PROGRESS 或直接到终态。

IN_PROGRESS

设备正在执行任务，可多次更新进度。支持与 QUEUED 来回切换（暂停/恢复场景）。

特殊：QUEUED → SUCCEEDED

允许直接跳过 IN_PROGRESS。场景：设备检查发现已是目标状态。

Terminal States (不可逆)

SUCCEEDED

设备上报成功。OTA 场景：固件安装完成。

FAILED / REJECTED

FAILED = 可重试失败（网络、校验）。REJECTED = 不可重试（硬件不兼容）。

TIMED_OUT

超过 inProgressTimeoutInMinutes 或 stepTimeout 后 AWS IoT 自动设置。

CANCELED / REMOVED

管理员操作触发：cancel-job 或 delete-job-execution API。
